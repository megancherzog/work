## Overview

This repository comes out of a team that moved fast with weekly releases, multiple products, and a support queue that told returned quick results. The samples are accurate, durable, and written for different audiences without relying on screenshots. UI changes every sprint, but well-written steps usually do not.

My work involved people as much as writing. I needed subject matter expert reviews, review cycles, and to continue to build shared standards for the team. The writing is the visible part, but the coordination behind it takes effort.

I have spent most of my career writing for users in finance, human resources,  federal regulatory, state regulatory, and financial aid. These are users who needed to get tasks done quickly and correctly.

The Python script in this repository came out of a real problem. The standards and style guide compliance was inconsistent across the team, and writers manually checking other writers' work was taking up a lot of our time. The script started as a way to make review checklists more consistent and grew into something the team could actually run themselves. The script checks both DITA structure and writing standards automatically and outputs a report anyone on the team can read and implement.
